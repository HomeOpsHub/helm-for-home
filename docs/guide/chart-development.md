# Chart Development

Create new Helm charts that follow the Helm for Home conventions.

## Chart Structure

Every chart follows this standard layout:

```
charts/stable/my-app/
├── Chart.yaml          # Chart metadata
├── values.yaml         # Default configuration
├── README.md           # Auto-generated by helm-docs
├── README.md.gotmpl    # Template for helm-docs (optional)
└── templates/
    ├── _helpers.tpl     # Template helpers
    ├── deployment.yaml
    ├── service.yaml
    ├── ingress.yaml
    ├── ingressroute.yaml
    ├── pvc.yaml
    ├── serviceaccount.yaml
    ├── hpa.yaml
    └── tests/
        └── test-connection.yaml
```

## Create a New Chart

### 1. Scaffold

```bash
# Start from an existing chart as a template
cp -r charts/stable/uptime-kuma charts/stable/my-app
```

### 2. Update Chart.yaml

```yaml
apiVersion: v2
name: my-app
description: A Helm chart for My App — a brief description of what it does
type: application
version: 0.1.0
appVersion: "1.0.0"
home: https://github.com/author/my-app
sources:
  - https://github.com/author/my-app
keywords:
  - my-app
  - self-hosted
maintainers:
  - name: HomeOpsHub
    url: https://github.com/HomeOpsHub
annotations:
  category: Utilities
```

### 3. Configure values.yaml

Set the container image and port:

```yaml
image:
  repository: author/my-app
  pullPolicy: IfNotPresent
  tag: ""  # Defaults to appVersion

service:
  type: ClusterIP
  port: 8080

persistence:
  enabled: false
  mountPath: /data
  size: 1Gi
  accessModes:
    - ReadWriteOnce
```

### 4. Update Templates

Adjust the deployment template for your app's specific needs (ports, env vars, volume mounts).

### 5. Generate Documentation

```bash
# Install helm-docs
brew install norwoodj/tap/helm-docs  # macOS
# or download from https://github.com/norwoodj/helm-docs/releases

# Generate README.md
helm-docs -c charts/stable/my-app
```

## Versioning Rules

- **Always** bump `version` in Chart.yaml for every change (follow [SemVer](https://semver.org/))
- Update `appVersion` when the upstream application version changes
- Patch bump (0.1.0 → 0.1.1): Bug fixes, minor template tweaks
- Minor bump (0.1.0 → 0.2.0): New features, new template resources
- Major bump (0.1.0 → 1.0.0): Breaking changes to values.yaml schema

## Testing

### Lint

```bash
helm lint charts/stable/my-app
```

### Template Rendering

```bash
helm template my-app charts/stable/my-app --debug
```

### Chart Testing (ct)

```bash
# Lint changed charts
ct lint --target-branch main

# Install and test on a Kind cluster
kind create cluster
ct install --target-branch main
```

## Stable vs Beta

| Tier | Directory | Requirements |
|------|-----------|-------------|
| **Stable** | `charts/stable/` | Passes lint, tested, documented |
| **Beta** | `charts/beta/` | Experimental, may have breaking changes |

New charts can start in `beta/` and graduate to `stable/` once validated.
